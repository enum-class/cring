Detailed Description

Represents the executor for managing asynchronous tasks in the Cring library.

The Executor structure holds information about the execution context, the current task being processed, the total number of tasks, the maximum capacity of tasks, and an array of Frame pointers representing individual tasks or coroutines.

  • struct IOContext ioc: The IOContext associated with the executor for managing I/O operations.
  • size_t current: Index of the currently active task in the executor.
  • size_t size: Current number of tasks scheduled in the executor.
  • size_t capacity: Maximum number of tasks the executor can handle.
  • struct Frame **frames: Dynamic array of Frame pointers representing individual tasks.

This structure plays a crucial role in orchestrating and managing the asynchronous execution of tasks within the Cring event loop.
